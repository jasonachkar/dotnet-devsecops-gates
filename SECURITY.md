# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Open a [GitHub Security Advisory](../../security/advisories/new)** on this repository.
2. Alternatively, open a GitHub Issue describing the concern (avoid including exploit details publicly).

Please include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact assessment

We will acknowledge your report within 48 hours and provide a timeline for resolution.

## Scope

This is a **portfolio/demo repository**. It contains no real secrets, credentials, or production data.
The "intentionally vulnerable" code exists only in isolated demo branches and is clearly marked.

## Supported Versions

| Version | Supported |
|---------|-----------|
| main    | Yes       |
| demo/*  | No (demo only) |
